# Providers

Провайдеры нужны, чтобы приложение не зависело от одного источника музыки.
Текущий прототип использует только источники, которые могут дать полный
аудиофайл.

## Internet Archive audio

Основной источник прототипа.

- Дает поиск по открытому аудиокаталогу.
- Через metadata API можно найти mp3/ogg/flac/wav-файлы.
- Подходит для демонстрации полного воспроизведения без 30-секундных фрагментов.

## Audius

Дополнительный источник.

- Ориентирован на открытый музыкальный каталог.
- Может отдавать полный stream для доступных треков.
- Иногда блокируется Cloudflare из серверной среды, поэтому Internet Archive
  используется как основной источник.

## YouTube

Используется для расширения поиска.

- Если задан `YOUTUBE_API_KEY`, используется официальный YouTube Data API.
- Если ключ не задан, backend пробует публичные Invidious-compatible endpoints.
- Для YouTube-результатов приложение использует embed-плеер и ссылку на
  источник, не скачивая и не извлекая аудио.

## Текущее правило

В результатах показываются треки с прямым `streamUrl` или YouTube `embedUrl`.
Короткие фрагменты не используются как отдельный источник воспроизведения.

## Общий интерфейс

```ts
export interface MusicProvider {
  name: string;
  searchTracks(query: string, limit: number): Promise<ProviderTrack[]>;
  getTrack?(externalId: string): Promise<ProviderTrack | null>;
}
```
